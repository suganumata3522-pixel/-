# VE・合意形成ワークショップ ― 進行ガイド

ゼネコン構造設計部の**中堅社員**向け勉強会。
「躯体コストを1割削減せよ」という典型的な VE（Value Engineering）要求を題材に、
意匠・設備・施工・施主との**利害が対立する調整会議**をロールプレイで体験する。

正解のある計算ドリルではない。**妥当解が複数あり、トレードオフを抱えたまま合意に持ち込む**訓練である。

## 狙い ― 伸ばす4つの力との対応

| 伸ばす力 | この課題のどこで鍛えるか |
| --- | --- |
| 新しい課題へ**応用する力** | 既知の架構知識を、コスト制約という新しい軸で再構成し VE 案を出す |
| 状況に応じて**考える力** | 1案では1割に届かない。複数案の組合せと優先順位を状況判断する |
| 周囲と**調整しながら進める力** | 無柱を守りたい意匠、梁せいを嫌う設備、工期を守る施工との合意形成 |
| 根本を理解し**説明する力** | 「なぜ削れるか／なぜ削れないか」を非専門家の施主に定量的に説明する |

## 題材（概要）

地上6階・S造の事務所ビル（設計施工）。正面に **18m×18m 無柱・2層吹抜のアトリウム**を持つ。
施主から「総工事費圧縮のため、構造躯体で1割（約0.7億円）削れないか」と要求が出た。
ただし **アトリウムのデザインは維持・工期延伸不可・安全性厳守**という制約付き。

詳細は `01_case.md`。

## 全体の流れ

```
[個人ワーク]                [役割設定]            [調整会議]            [講評]
VE案を5つ以上出す   →   役割カード配布   →   合意形成ロールプレイ  →  相互講評＋解説
（30分）                  各自の利害を読む      （40〜60分）           （30分）
```

1. **個人ワーク**（`03_tasks.md` 課題①）：各自が VE 案を出し、削減効果と副作用を表にする
2. **役割設定**（`02_roles.md`）：意匠／設備／施工／構造／施主PM のカードを配る。**自分の役の非公開の利害**を読み込む
3. **調整会議**：施主の1割要求に対し、各役が立場を主張しながら合意案を作る
4. **講評**（`04_solutions.md`）：VE メニューの模範例・トレードオフ整理・評価ルーブリックで振り返る

## タイムテーブル例

| 区分 | 半日版（3.5h） | 短縮版（2h） |
| --- | --- | --- |
| 趣旨説明・与条件読込 | 20分 | 15分 |
| 個人ワーク（VE案出し） | 40分 | 25分 |
| 役割カード読込・作戦 | 20分 | 10分 |
| 調整会議 | 60分 | 40分 |
| 合意案の発表 | 30分 | 15分 |
| 講評・解説 | 40分 | 15分 |

人数：1チーム5〜6名（5役＋オブザーバ）。複数チームで並行し、合意案を比較すると学びが深い。

## 評価の考え方

**答えの一致を採点しない。** 評価軸は次の4点（詳細ルーブリックは `04_solutions.md`）。

- 応用：VE 案の引き出しの数と質（数量VE／コストVE／施工VE を区別できているか）
- 判断：トレードオフを定量で比較し、優先順位をつけて取捨できたか
- 調整：相手の**隠れた制約**を引き出し、Win-Win の代替案を作れたか
- 説明：施主（非専門家）に削減根拠と安全性を1枚で説明できたか
