## 1. サイトの全体像を知る

このサイトは **HTMLファイルだけでできた「静的サイト」** です。WordPressのような管理画面・データベース・サーバープログラムは一切不要で、ファイルを置くだけで動きます。壊れにくく、表示が速く、無料で公開できるのが利点です。

### ファイル構成

```
kouzou-site/
├── index.html                      ← トップページ
├── about.html                      ← このサイトについて(免責事項つき)
├── privacy.html                    ← プライバシーポリシー(AdSense申請に必須)
├── css/
│   └── style.css                   ← サイト全体のデザイン(色・レイアウト)
├── js/
│   └── main.js                     ← スマホ用メニューの動き
├── articles/                       ← ★記事はすべてこのフォルダに入れる
│   ├── danmen-niji-moment.html     (断面二次モーメント)
│   ├── tawami.html                 (梁のたわみ)
│   ├── rahmen-brace.html           (ラーメン構造とブレース構造)
│   └── kyoyo-oryokudo.html         (許容応力度計算)
└── tools/
    └── section-calculator.html     ← 断面性能計算ツール
```

### 覚えておくべきことは3つだけ

| やりたいこと | 触るファイル |
|---|---|
| 記事を増やす | `articles/` に新しいHTMLファイルを追加 + `index.html` にカードを追加 |
| 文章を直す | 該当するHTMLファイルの本文部分 |
| デザイン(色など)を変える | `css/style.css` の先頭にある色の設定 |

## 2. サイトを自分のパソコンで見る

公開前でも、ファイルをパソコンに置けばブラウザで見られます。

### 手順(ZIPでダウンロードする方法)

1. ブラウザで自分のGitHubリポジトリ(`github.com/suganumata3522-pixel/-`)を開く
2. 画面左上のブランチ名(`main` と書かれたプルダウン)をクリックし、**`claude/beautiful-babbage-od9n55`** を選ぶ
   ※サイトのファイルはまだこの「作業ブランチ」にあります。第3章でmainに合流させます
3. 緑色の **「Code」ボタン → 「Download ZIP」** をクリック
4. ダウンロードしたZIPを右クリック →「すべて展開」(Macはダブルクリック)
5. 展開したフォルダの中の `kouzou-site` フォルダを開き、**`index.html` をダブルクリック**

これだけでブラウザにトップページが表示されます。リンクも計算ツールもすべてそのまま動きます。

> **ポイント**: HTMLファイルは「メモ帳で開けば編集」「ブラウザで開けば閲覧」になります。ダブルクリックで編集画面が開いてしまう場合は、右クリック →「プログラムから開く」→ Chrome や Edge を選んでください。

## 3. サイトをインターネットに公開する

> **【2026年6月追記】公開方式は「gh-pagesブランチ」方式に決定済みです**
>
> 公開用ブランチ(gh-pages)の準備はClaudeが自動で行っています。あなたがやるのは初回の2クリックだけです:
>
> 1. GitHubのリポジトリ → **Settings → Pages** を開く
> 2. Source を「**Deploy from a branch**」、Branch を「**gh-pages**」+「**/ (root)**」にして **Save**
>
> 数分後に **`https://suganumata3522-pixel.github.io/-/`** でサイトが公開されます。
>
> **記事を追加・修正した後の反映方法**: Claudeに「公開ブランチを更新して」と頼むだけです(mainの最新内容がgh-pagesに反映されます)。
>
> 以下の「新しい専用リポジトリを作る」手順は、将来きれいなURLにしたい場合の代替手段として残してあります。なお独自ドメイン(ステップ4)を設定すれば、現在の方式のままURLの見た目も解消できます。

公開には GitHub の無料機能 **「GitHub Pages」** を使います。サーバー代は無料です。

### ステップ1: 作業ブランチを main に合流させる(マージ)

今のサイトファイルは作業用ブランチに入っているので、まず本流(main)に取り込みます。

1. GitHubのリポジトリページを開く
2. 上部タブの **「Pull requests」→ 緑の「New pull request」** をクリック
3. `base: main` ← `compare: claude/beautiful-babbage-od9n55` となるように選ぶ
4. 緑の **「Create pull request」** → もう一度 **「Create pull request」**
5. 緑の **「Merge pull request」→「Confirm merge」** をクリック

これで main ブランチにサイト一式が入りました。
(※Claudeに「プルリクエストを作ってマージまでして」と頼むこともできます)

### ステップ2: サイト専用の新しいリポジトリを作る(推奨)

GitHub Pagesは「リポジトリの一番上の階層」か「docsフォルダ」しか公開できません。今のリポジトリには別のプロジェクトも入っているため、**サイト専用のリポジトリを新しく作るのが一番きれいで簡単**です。

1. GitHub右上の **「+」→「New repository」** をクリック
2. Repository name に `kouzou-navi` などの名前を入力(これがURLの一部になります)
3. 「Public」を選んで **「Create repository」**
4. 作成直後の画面にある **「uploading an existing file」** のリンクをクリック
5. 第2章でダウンロード・展開した `kouzou-site` フォルダを開き、**中身を全部選択**して(index.htmlやarticlesフォルダなど)、ブラウザの点線枠に**ドラッグ&ドロップ**
6. 下の緑ボタン **「Commit changes」** をクリック

> **重要**: `kouzou-site` フォルダごとではなく、**フォルダの中身**をアップロードしてください。`index.html` がリポジトリの一番上に来るのが正解です。

### ステップ3: GitHub Pages をオンにする

1. 新しいリポジトリ(`kouzou-navi`)の **「Settings」タブ → 左メニューの「Pages」** を開く
2. 「Source」を **「Deploy from a branch」** にする
3. Branch を **「main」**、フォルダを **「/ (root)」** にして **「Save」**
4. 1〜3分待ってページを再読み込みすると、上部に公開URLが表示されます:
   **`https://suganumata3522-pixel.github.io/kouzou-navi/`**

このURLにアクセスすれば、世界中の誰でもサイトを見られます。

### ステップ4(任意・収益化するなら推奨): 独自ドメインを設定する

`kouzou-navi.com` のような独自ドメインがあると、AdSense審査・SEO・信頼性で有利です。

1. お名前.com / ムームードメイン / Cloudflare Registrar などでドメインを購入(年1,000〜2,000円程度)
2. ドメイン会社の設定画面(DNS設定)で **CNAMEレコード** を追加:
   - ホスト名: `www`、値: `suganumata3522-pixel.github.io`
3. GitHubのリポジトリ → Settings → Pages → 「Custom domain」に `www.kouzou-navi.com` を入力して Save
4. 「Enforce HTTPS」にチェック(表示されるまで数十分かかることがあります)

> 設定で詰まったら、画面に出ているエラーをそのままClaudeに伝えてください。

## 4. サイトを編集する3つの方法

### 方法A: Claude(このチャット)に頼む【一番かんたん・推奨】

やりたいことを日本語で伝えるだけです。例:

- 「『せん断力図の描き方』という記事を追加して」
- 「トップページのキャッチコピーを○○に変えて」
- 「サイトの色をもう少し落ち着いたグレー系にして」
- 「計算ツールにH形鋼の断面も追加して」

Claudeがファイルを編集してGitHubに保存(プッシュ)するところまで行います。

### 方法B: GitHubのウェブ画面で直接編集する【インストール不要】

ちょっとした誤字修正などに便利です。

1. GitHubでリポジトリを開き、編集したいファイル(例: `articles/tawami.html`)をクリック
2. ファイル表示画面の右上にある **えんぴつアイコン(Edit this file)** をクリック
3. テキストを編集する
4. 右上の緑ボタン **「Commit changes...」** をクリック
5. 変更内容のメモ(例:「誤字修正」)を書いて **「Commit changes」**

公開済みサイトなら、1〜2分後に自動で反映されます。

### 方法C: パソコンで編集してアップロードする【まとめて作業向き】

1. 無料エディタ **VS Code**(code.visualstudio.com)をインストール
2. 第2章の方法でファイルをダウンロードし、VS Codeでフォルダごと開く
3. 編集して保存 → ブラウザで `index.html` を開いて確認(編集→保存→ブラウザ再読み込み、を繰り返す)
4. 完成したら、GitHubのリポジトリ画面で **「Add file」→「Upload files」** から該当ファイルをアップロード(同名ファイルは上書きされます)

## 5. よくある編集箇所と書き換え方

HTMLは「タグ(`<タグ名>〜</タグ名>`)で文章を挟む」だけの仕組みです。**タグの中の日本語を書き換える**のが基本で、タグ自体(`<` や `>` の記号)を消さないよう注意してください。

### サイト名を変えたい

全ページのヘッダーにあります(各HTMLファイルで同じ修正が必要):

```html
<span>構造設計ナビ<small>STRUCTURAL DESIGN NAVI</small></span>
```

### トップページのキャッチコピーを変えたい

`index.html` の `<section class="hero">` 内:

```html
<h1>建築の<span class="accent">構造設計</span>を、<br>基礎から実務までわかりやすく。</h1>
<p>構造力学の基本公式から、…</p>
```

`<br>` は改行、`<span class="accent">〜</span>` で挟んだ部分は水色になります。

### サイトの色を変えたい

`css/style.css` の先頭にまとまっています。ここを変えるだけで全ページに反映されます:

```css
:root {
  --navy: #1a3a5c;        /* ヘッダーや見出しの紺色 */
  --accent: #e8843c;      /* ボタンや強調のオレンジ */
  --accent-soft: #fdf1e7; /* オレンジの淡色(タグの背景など) */
  --bg-soft: #f7f9fb;     /* ヒーローやボックスの薄いグレー */
}
```

`#122d4d` のような記号は「カラーコード」です。「カラーコード 一覧」で検索すると選べます。

### 「お問い合わせ」を実際に動くようにしたい

無料の **Googleフォーム** が簡単です。フォームを作成 →「送信」→ リンクをコピーし、`about.html` の以下を書き換えます:

```html
<p>記事内容の誤りのご指摘…は、<a href="https://forms.gle/あなたのフォームID">お問い合わせフォーム</a>よりご連絡ください。</p>
```

## 6. 記事を追加する手順

**「既存の記事をコピーして中身を入れ替える」** のが最も確実な方法です。

### ステップ1: テンプレートにする記事を決めてコピーする

`articles/tawami.html` が標準的な構成なのでおすすめです。

- **GitHub上で行う場合**: ファイルを開き → えんぴつアイコン → 全文を選択してコピー → `articles` フォルダで「Add file」→「Create new file」→ ファイル名を入力 → 貼り付け
- **パソコンで行う場合**: ファイルをコピーして名前を変える

**ファイル名のルール**: 半角英数字とハイフンのみ。記事の内容を表す英語/ローマ字にします。
例: `oryokudo.html`(応力度)、`za-kutsu.html`(座屈)、`hari-keisan.html`(梁の計算)

### ステップ2: 5か所を書き換える

新しいファイルの中で、以下を上から順に書き換えます。

**① ページタイトル(検索結果に出る題名)**
```html
<title>座屈とは？オイラーの公式と座屈長さをわかりやすく解説｜構造設計ナビ</title>
```

**② meta description(検索結果に出る説明文・100字前後)**
```html
<meta name="description" content="座屈は細長い圧縮材が横に膨らんで…">
```

**③ パンくずリスト(ページ上部の現在地表示)**
```html
<nav class="breadcrumb"><a href="../index.html">ホーム</a> › 構造力学の基礎 › 座屈</nav>
```

**④ 記事の見出しと日付**
```html
<span class="tag">構造力学の基礎</span>
<span>更新日：2026年6月20日</span>
...
<h1>座屈とは?オイラーの公式と座屈長さをわかりやすく解説</h1>
```

**⑤ 本文**(`<h1>` の下から `</article>` の手前まで)を書き換える。使えるパーツは次のとおり:

```html
<h2>大見出し(青い帯つき)</h2>
<h3>小見出し(下線つき)</h3>
<p>段落。文章はこのタグで挟みます。</p>
<p><strong>太字にしたい部分</strong>はこう書きます。</p>

<ul>
  <li>箇条書き1</li>
  <li>箇条書き2</li>
</ul>

<!-- 数式ボックス(水色) -->
<div class="formula">
  P = π²EI / L²
  <span class="note">P：座屈荷重 ／ L：座屈長さ</span>
</div>

<!-- POINTボックス(黄色) -->
<div class="point-box">
  <span class="label">POINT</span>
  <p>ここに要点を書く。</p>
</div>

<!-- 補足ボックス(緑) -->
<div class="note-box">
  <span class="label">あわせて確認</span>
  <p>関連記事は「<a href="tawami.html">梁のたわみ</a>」をどうぞ。</p>
</div>

<!-- 表 -->
<table class="data">
  <tr><th>見出し1</th><th>見出し2</th></tr>
  <tr><td>データ1</td><td>データ2</td></tr>
</table>
```

### ステップ3: トップページに記事カードを追加する

`index.html` の「人気の記事」の `<div class="article-grid">` 内に、既存カードをコピーして追加します:

```html
<a href="articles/za-kutsu.html" class="article-card">
  <div class="thumb thumb-teal">🏛️</div>
  <div class="body">
    <span class="tag">構造力学の基礎</span>
    <h3>座屈とは？オイラーの公式と座屈長さをわかりやすく解説</h3>
    <p>細長い柱はなぜ折れる前に曲がるのか。座屈の基本を図解で。</p>
  </div>
</a>
```

サムネイルの色は `thumb-blue` / `thumb-teal` / `thumb-orange` / `thumb-purple` の4種類、絵文字は自由に変えられます。

### ステップ4: 既存記事のサイドバーに相互リンクを追加する

関連する既存記事の「関連記事」ウィジェットに、新記事へのリンクを1行追加します:

```html
<li><a href="za-kutsu.html">座屈とは？オイラーの公式と座屈長さ</a></li>
```

内部リンクの張り合いはSEO(検索順位)に大きく効きます。**新記事を1本書いたら、関連する既存記事2〜3本からリンクを張る**のを習慣にしてください。

### ステップ5: サイト内検索に新記事を登録する

トップページの検索窓は `js/main.js` の記事リストを検索しています。`var ARTICLES = [` の中に1行追加してください:

```js
{ title: '座屈とは？オイラーの公式と座屈長さ', url: 'articles/za-kutsu.html', keywords: '座屈 オイラー 座屈長さ 圧縮' },
```

`keywords` には、読者が検索しそうな言葉をスペース区切りで自由に入れます。

## 9. 困ったときのQ&A

**Q. 編集したのに公開サイトに反映されない**
A. GitHub Pagesの反映には1〜3分かかります。それでも変わらない場合はブラウザのキャッシュが原因なので、`Ctrl + F5`(Macは `Cmd + Shift + R`)で強制再読み込みしてください。

**Q. ページのレイアウトが突然崩れた**
A. ほぼ確実にタグの閉じ忘れ(`</div>` の消しすぎ等)です。直前に編集したファイルをGitHubの「History」から開き、「Revert」または以前の版をコピーして戻せます。崩れたファイル名をClaudeに伝えて「直して」でもOKです。

**Q. 日本語が文字化けする**
A. ファイルを保存するときの文字コードが原因です。メモ帳なら「名前を付けて保存」で文字コードを **UTF-8** にしてください(VS Codeなら最初からUTF-8です)。

**Q. スマホでの見え方を確認したい**
A. Chromeでサイトを開き `F12` → 左上のスマホアイコン(デバイスツールバー)をクリックすると、スマホ画面をシミュレートできます。

**Q. 記事を消したい**
A. GitHubでそのファイルを開き、右上の「…」→「Delete file」。あわせて `index.html` のカードと、他記事のサイドバーのリンクも削除してください。

**Q. 操作を間違えてぐちゃぐちゃになった**
A. GitHubにはすべての履歴が残っているので、何をしても元に戻せます。慌てずに、リポジトリの「History」(時計マークのアイコン)から戻したい時点を探すか、Claudeに「○日前の状態に戻して」と伝えてください。
