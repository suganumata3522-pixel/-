# 電脳せどり管理ツール

利益商品の自動リサーチから販路設定、購入日の決定判断、仕入・売上管理までを
一元管理できるWebアプリです(Flask + SQLite、ローカル動作)。

## 機能

| 機能 | 内容 |
| --- | --- |
| 📊 ダッシュボード | 今月の売上・純利益・仕入額・在庫・候補数、月次推移、次のおすすめ購入日 |
| 🔍 商品リサーチ | 楽天市場 / Yahoo!ショッピングAPIを横断検索(売れ筋順・仕入価格の下限/上限指定・最大50件)し、販路手数料・ポイント還元込みで利益・利益率・ROIを自動試算 |
| 🏷️ Amazon売価の自動取得 | JANコードからKeepa API / Amazon SP-APIでAmazon.co.jpのカート価格・売れ筋ランキングを取得し、想定売価として自動採用(結果はキャッシュ) |
| 🤖 自動リサーチ | 保存キーワードを一括検索し、利益条件(最低利益額・利益率・ROI)を満たす商品を自動で候補登録。Amazon相場が確認できた商品だけに絞る「相場必須モード」と売れ筋ランキング上限で、売価の裏付けのない商品を除外。推奨購入日も自動付与 |
| 📅 購入日判断 | 5と0のつく日・5のつく日・スーパーセール/お買い物マラソン時期などのカレンダーから、最も実質仕入額を下げられる購入日を推奨 |
| 🏪 販路設定 | Amazon(FBA/自己発送)・メルカリ・ヤフオク!・ラクマ・Yahoo!フリマなどの手数料体系を自由に設定 |
| 📦 仕入管理 | 候補からワンクリックで仕入登録。数量・獲得ポイント・状態(在庫/出品中/売却済)を管理 |
| 📝 出品作成 | 販路別テンプレートでタイトル・説明文・タグ・推奨価格(損益分岐価格付き)を自動生成。複数販路へ一括作成、コピーボタンで出品作業を時短。Claude APIキーを登録すればAIで文面を磨き上げ |
| 📷 出品画像の整形 | 商品写真を各販路の推奨仕様(白背景・正方形・規定サイズ)へ自動変換。スマホ写真の向きも自動補正 |
| 🔔 価格改定アラート | 在庫商品のAmazon相場をワンクリックで再取得し、しきい値以上の値下がりをダッシュボードに警告 |
| 📱 バーコード仕入れ | スマホカメラでJANをスキャン(または直接入力)→ Amazon相場と利益を即計算、その場で候補登録 |
| 💴 資金繰り | 月間仕入予算の消化率・残額、在庫資金、利益目標の達成率を一画面で管理 |
| 🧾 経費管理 | 梱包材・送料・月額利用料などを記録。レポートで営業利益(粗利-経費)を自動計算、CSV出力対応 |
| 📈 価格履歴グラフ | 候補商品の90日価格推移をチャート表示。「たまたま高いだけ」の商品への飛びつきを防止 |
| 🚀 SP-API自動出品 | Amazon出品ドラフトをワンクリックでSP-API送信(既存ASINへの相乗り出品) |
| 🔄 在庫連動 | 売り切れ時にAmazonは在庫0を自動送信、APIのない販路は「要取下げ」警告で二重販売を防止 |
| ⏰ 再出品リマインダー | 出品からN日経過した商品を検知し、ワンクリック複製で新着上位への出し直しを支援 |
| 💰 売上管理 | 売却登録時に販路手数料・ポイント按分原価を自動計算して純利益を算出 |
| 📈 レポート | 月次の売上・仕入・純利益・利益率、販路別実績、CSVエクスポート(Excel対応) |

📖 **[かんたん操作マニュアル(スクリーンショット付き)](docs/MANUAL.md)** — はじめての方はこちら

## セットアップ

**かんたん起動(おすすめ)** — Pythonをインストール済みなら、ダブルクリックするだけ:

- Windows: `start_windows.bat`
- Mac: `start_mac.command`

初回は自動で必要な部品(.venv)をインストールし、ブラウザが自動で開きます。

**コマンドで起動する場合:**

```bash
pip install -r requirements.txt
python app.py
```

ブラウザで http://127.0.0.1:5050 を開きます。
データは `sedori.db`(SQLite)に保存されます。

## APIキーの設定

初期状態では**デモデータ**で動作します。実データを検索するには「設定」画面で
APIキーを登録してください。

- **楽天市場**: [楽天ウェブサービス](https://webservice.rakuten.co.jp/) でアプリIDを取得(無料)
- **Yahoo!ショッピング**: [Yahoo!デベロッパーネットワーク](https://developer.yahoo.co.jp/) でClient IDを取得(無料)

### Amazon売価の自動取得(どちらか一方でOK、Keepa優先)

- **Keepa**: [keepa.com/#!api](https://keepa.com/#!api) でAPIキーを取得(有料サブスクリプション)。
  JANから1リクエストで価格・ランキングが取れるため推奨。
  📖 **[Keepa APIキー取得ガイド(画面付き)](docs/KEEPA_API_GUIDE.md)**
- **Amazon SP-API**: Amazon出品者アカウントで
  [SP-APIアプリを登録](https://developer-docs.amazon.com/sp-api/)し、
  LWAクライアントID / クライアントシークレット / リフレッシュトークンを設定。
  JAN→ASIN解決(カタログAPI)後にPricing APIでカート価格を取得します。

取得結果はSQLiteにキャッシュされ(既定24時間、設定で変更可)、トークン消費と
レート制限を抑えます。どちらも未設定の場合は「仕入価格×係数」の推定にフォールバックします。
JANが取れない商品(楽天など)は、リサーチ画面の「JANコードからAmazon相場を単体検索」で
手動確認できます。

## 使い方の流れ

1. **販路設定** — 自分が使う販路の手数料・送料を確認/調整
2. **設定** — APIキー、利益条件(最低利益額/利益率/ROI)、想定売価係数を設定
3. **商品リサーチ** — キーワード検索 or 保存キーワードの一括自動リサーチ
4. **候補商品** — 利益条件を満たした商品を確認し「仕入予定」へ
5. **購入日判断** — 推奨購入日(ポイントアップ日)を確認して仕入実行
6. **仕入管理 → 売上管理** — 仕入登録 → 売却登録で純利益を自動計算
7. **レポート** — 月次・販路別の実績を確認、CSVで出力

## 注意事項

- 想定売価はAmazon相場が取得できた場合はカート価格、取得できない場合は
  「仕入価格 × 係数(初期値1.35)」の概算です。係数推定の商品は候補登録時に
  想定売価を上書きしてください。
- セールイベントのうち「予測」表示のもの(お買い物マラソン等)は開催傾向からの
  推定です。必ず公式の開催情報を確認してください。
- 各モールのAPI利用規約・出品規約を遵守してください。
